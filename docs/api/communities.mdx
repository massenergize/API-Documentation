### /communities.list

- Optional: `zipcode:str`, `max_distance:int`

This route is used to list active communities. With optional arguments, it will filter active communities based on how far away from the given `zipcode`

### /communities.update

<Highlight level={1}>Admins Only</Highlight>

- Required: `community_id:int`

Refer to [Community()](global-variables#community) for the full list of fields that can be updated.

### /communities.delete, /communities.remove

<Highlight level={1}>Super Admins Only</Highlight>

- Required: `id:int`

### /communities.join

<Highlight level={0}>Login Required</Highlight>

- Required: `community_id:int`

This route is used to add the current user to the specified community.

### /communities.leave

<Highlight level={0}>Login Required</Highlight>

- Required: `community_id:int`

This route is used to remove the current user in the specified community, if existed.

### /communities.custom.website.add

<Highlight level={1}>Admins Only</Highlight>

- Required: `website:str`, (`subdomain:str` or `community_id:int`)

### /communities.actions.completed

- Optional: `community_id:int`, `subdomain:str`, `communities:str_list`, `actions:str_list`, `time_range:str`, `end_data:str`, `start_date:str`

This route is used to list actions completed by members of a community or team. Include any actions which are DONE or TODO (so not really completed).
